Library Management System

Welcome to the Library Management System. This project is built with Django, Django Rest Framework (DRF), and Dramatiq for background tasks. Below is an overview of the project's components and their functionality.

Components

Models

Defines the data structure of the application:

  • Author: Stores information about book authors.
  • Publisher: Stores information about book publishers.
  • Category: Stores categories to classify books.
  • Book: Stores details about books, linked to authors, publishers, and categories.
  • Member: Stores information about library members.
  • Borrow: Tracks borrowing records of books by members.

Serializers

Convert model instances to JSON format and vice versa for API interactions:

  • AuthorSerializer
  • PublisherSerializer
  • CategorySerializer
  • BookSerializer
  • MemberSerializer
  • BorrowSerializer

Viewsets

Handle API requests for CRUD operations:

  • AuthorViewSet
  • PublisherViewSet
  • CategoryViewSet
  • BookViewSet
  • MemberViewSet
  • BorrowViewSet

Admin Interface

Provides an interface to manage data in a user-friendly way:

  • Registered models: Author, Publisher, Category, Book, Member, Borrow

Templates

Render HTML responses. In this case, the home page template.

Filters

Allow easy filtering of querysets based on criteria defined in the filter classes:

  • BookFilter

Celery

Handles background tasks asynchronously:

  • Configured in celery.py

Dramatiq

Another library used for background tasks:

  • Tasks defined in tasks.py

Redis

Used as the message broker for Celery and Dramatiq to manage the task queue.

How to Use the Project

  1. Set Up the Environment: Create and activate a virtual environment, install dependencies, and set up the database.
  2. Run Redis: Start the Redis server, which is used by Celery and Dramatiq for task management.
  3. Run the Celery Worker: Start the Celery worker to process background tasks.
  4. Run the Django Development Server: Start the Django development server to serve the application.
  5. Access the Admin Interface: Navigate to http://localhost:8000/admin to manage the data.
  6. Interact with the API: Use tools like Postman or a web browser to interact with the API endpoints.
